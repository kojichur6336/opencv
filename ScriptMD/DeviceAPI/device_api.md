## 1. `Device.name()`

### 1.1 接口说明

`Device.name()` 用于 **获取或设置用户自定义的设备名称**。

该名称对应系统路径：

> 设置 > 通用 > 关于本机 > 名称

该接口在 Lua 中采用 **getter / setter 合并设计**：

- **不传参数**：获取当前设备名称
- **传入字符串参数**：设置新的设备名称

---

### 1.2 参数说明

> 说明：该接口使用 **位置参数**，不使用 key-value 形式。

| 参数位置 | 类型   | 是否必选 | 中文说明       | 示例            |
|----------|--------|----------|----------------|-----------------|
| 第 1 个  | string | 否       | 新的设备名称   | "MyNewiPhone" |

---

### 1.3 返回值说明

| 调用形式            | 返回值个数 | 返回类型 | 中文说明     |
|---------------------|------------|----------|--------------|
| `Device.name()`     | 1          | string   | 当前设备名称 |
| `Device.name(name)` | 0          | 无       | 无返回值     |

---

### 1.4 Lua 调用示例

```lua
-- 获取设备名称
local name = Device.name()
Log.print('当前设备名称: ' .. tostring(name))

-- 设置设备名称
Device.name('MyNewiPhone')
Log.print('已尝试设置设备名称')
```

## 7. `Device.telephonyInfo()`

### 7.1 接口说明

`Device.telephonyInfo()` 用于获取设备底层 **移动通信与硬件标识信息**，支持 **单卡 / 双卡设备**。

该接口同时支持两种使用模式：

- **全量模式**：不传参数，返回所有卡槽的完整信息结构
- **智能模式**：按需获取指定字段（如 IMEI、ICCID），并可指定卡槽索引

适用于读取 IMEI、IMSI、ICCID、MEID、CSN 等基带相关标识信息。

---

### 7.2 参数说明

> 说明：该接口使用 **位置参数**，不使用 key-value 形式。

| 参数位置 | 类型   | 是否必选 | 中文说明                     | 示例        |
|----------|--------|----------|------------------------------|-------------|
| 第 1 个  | string | 否       | 硬件 / 通信字段名称          | "IMEI"    |
| 第 2 个  | int    | 否       | 卡槽索引（从 1 开始）        | 1 / 2       |

#### 参数行为说明

- **不传任何参数**：返回全量通信信息 table
- **仅传第 1 个参数**：返回主卡（Slot 1）对应字段值
- **传第 1、2 个参数**：返回指定卡槽对应字段值

---

### 7.3 返回值说明

| 调用形式                               | 返回值个数 | 返回类型 | 中文说明                         |
|----------------------------------------|------------|----------|----------------------------------|
| `Device.telephonyInfo()`               | 1          | table    | 所有卡槽的通信信息集合           |
| `Device.telephonyInfo(field)`          | 1          | string   | 主卡指定字段的值                 |
| `Device.telephonyInfo(field, slot)`    | 1          | string   | 指定卡槽的字段值                 |

> 注意：
> - 若字段不存在或无法获取，返回值为字符串 `"N/A"`
> - 在权限不足或环境不支持时，可能返回 `nil`

---

### 7.4 常见字段说明

| 字段名 | 类型   | 中文说明                         |
|--------|--------|----------------------------------|
| IMEI   | string | 国际移动设备识别码               |
| IMSI   | string | 国际移动用户识别码               |
| ICCID  | string | SIM 卡物理卡号                   |
| MEID   | string | 移动设备识别码（CDMA 制式）      |
| CSN    | string | SIM / 卡片序列号                 |

---

### 7.5 Lua 调用示例

#### 示例 1：智能模式（按需读取）

```lua
local imei0 = Device.telephonyInfo('IMEI')        -- 默认主卡 (Slot 1)
local imei1 = Device.telephonyInfo('IMEI', 1)     -- 指定副卡 (Slot 2)

Log.print('[智能模式] 主卡 IMEI: ' .. tostring(imei0))
Log.print('[智能模式] 副卡 IMEI: ' .. tostring(imei1))
```

---

#### 示例 2：全量模式（结构化遍历）

```lua
local info = Device.telephonyInfo()

if type(info) == 'table' then
    Log.print('开始全量读取移动通讯信息...')
    Log.print('------------------------------------')

    -- 第一层循环：遍历卡槽 (Slot_1, Slot_2)
    for slotName, details in pairs(info) do
        Log.print('[' .. slotName .. ' 详细列表]:')

        -- 第二层循环：遍历该卡槽下的所有硬件参数
        if type(details) == 'table' then
            for key, value in pairs(details) do
                Log.print('  ' .. key .. ': ' .. tostring(value))
            end
        else
            Log.print('  错误: 该槽位数据格式不是 table')
        end

        Log.print('------------------------------------')
    end

    Log.print('全量信息读取完毕。')
else
    Log.print('错误: Device.telephonyInfo() 未能返回 table 对象')
end
```

## 8. `Device.ip()`

### 8.1 接口说明

`Device.ip()` 用于获取设备当前的 **IP 地址信息**，支持 **Wi-Fi / 蜂窝网络**，并可一次性获取 **所有网络接口**。

该接口支持多种获取模式，适用于网络诊断、环境识别、日志上报等场景。

---

### 8.2 参数说明

> 说明：该接口使用 **位置参数**，不使用 key-value 形式。

| 参数位置 | 类型   | 是否必选 | 中文说明                     | 示例        |
|----------|--------|----------|------------------------------|-------------|
| 第 1 个  | string | 否       | 网络类型选择                 | "wifi" / "cellular" / "all" |

#### 参数行为说明

- **不传参数**：默认获取 Wi-Fi IP（等同于 `"wifi"`）
- **传 `"wifi"`**：获取 Wi-Fi 网络 IP（通常为 `en0`）
- **传 `"cellular"`**：获取蜂窝移动网络 IP
- **传 `"all"`**：返回所有网络接口及其 IP 地址的 table

---

### 8.3 返回值说明

| 调用形式                 | 返回值个数 | 返回类型 | 中文说明                           |
|--------------------------|------------|----------|------------------------------------|
| `Device.ip()`            | 1          | string   | Wi-Fi 网络 IP 地址                 |
| `Device.ip("wifi")`      | 1          | string   | Wi-Fi 网络 IP 地址                 |
| `Device.ip("cellular")`  | 1          | string   | 蜂窝移动网络 IP 地址               |
| `Device.ip("all")`       | 1          | table    | 所有网络接口与 IP 的映射表         |

> 注意：
> - 若指定网络类型不存在，可能返回 `nil`
> - `"all"` 模式下仅返回当前可用的网络接口

---

### 8.4 `all` 模式返回结构说明

当调用 `Device.ip("all")` 时，返回值为一个 **Lua table**：

- `key`：网络接口名称（如 `en0`, `pdp_ip0`）
- `value`：对应接口的 IP 地址字符串

#### 示例结构

```lua
{
    en0 = "192.168.1.10",
    pdp_ip0 = "10.23.45.6"
}
```

---

### 8.5 Lua 调用示例

#### 示例 1：默认模式（Wi-Fi）

```lua
local wifiIP = Device.ip()
Log.print("默认 IP (Wi-Fi): " .. tostring(wifiIP))
```

---

#### 示例 2：获取蜂窝网络 IP

```lua
local cellularIP = Device.ip("cellular")
Log.print("蜂窝网络 IP: " .. tostring(cellularIP))
```

---

#### 示例 3：全量模式（遍历所有网络接口）

```lua
local all = Device.ip("all")

if type(all) == "table" then
    Log.print("开始读取所有网络接口 IP...")
    Log.print("------------------------------------")

    for name, ip in pairs(all) do
        Log.print("网卡: " .. tostring(name) .. " -> IP: " .. tostring(ip))
    end

    Log.print("------------------------------------")
    Log.print("网络接口信息读取完毕。")
else
    Log.print("错误: Device.ip('all') 未能返回 table 对象")
end
```

## 9. `Device.dpi()`

### 9.1 接口说明

`Device.dpi()` 用于获取设备屏幕的 **物理像素密度 (DPI/PPI)**，用于衡量屏幕每英寸包含的像素数量。  
Retina 屏幕通常为 326 或 460，不同设备型号略有差异。

适用于：图形计算、手势模拟、界面适配等场景。

---

### 9.2 参数说明

> 说明：该接口无参数。

| 参数位置 | 类型 | 是否必选 | 中文说明 | 示例 |
|----------|------|----------|----------|------|
| 无       | —    | 否       | 无参数  | —    |

---

### 9.3 返回值说明

| 调用形式        | 返回值个数 | 返回类型 | 中文说明                           |
|-----------------|------------|----------|------------------------------------|
| `Device.dpi()`  | 1          | number   | 当前设备屏幕的物理像素密度 (DPI)  |

> 示例：460 (iPhone 15 Pro)，264 (iPad Pro)

---

### 9.4 Lua 调用示例

#### 示例 1：获取屏幕物理 DPI

```lua
local myDpi = Device.dpi()
Log.print('当前设备 DPI: ' .. tostring(myDpi))
```

#### 示例 2：计算每厘米对应的像素

```lua
-- 1 英寸 = 2.54 厘米
local pxPerCm = myDpi / 2.54
Log.print('屏幕每厘米对应的像素值: ' .. string.format('%.2f', pxPerCm))
```

#### 示例 3：实际应用：模拟滑动 3 厘米

```lua
local slidePx = math.floor(pxPerCm * 3)
Log.print('如果要模拟滑动 3 厘米，需要移动的像素距离为: ' .. slidePx)
```

## 14. `Device.wifiAddress()`

### 14.1 接口说明

`Device.wifiAddress()` 用于获取设备 **Wi‑Fi 网卡的物理地址（MAC Address）**，  
返回格式示例：`A4:D1:8C:12:34:56`。

该地址为 **Wi‑Fi 芯片的真实硬件地址**，具有唯一性。

常用于：  
- 设备唯一性识别  
- 设备指纹 / 风控  
- 根据 MAC 前缀判断网卡厂商信息  

---

### 14.2 参数说明

该接口 **无需参数**。

| 参数位置 | 类型 | 是否必选 | 中文说明 | 示例 |
|----------|------|----------|----------|------|
| 无       | —    | —        | —        | —    |

---

### 14.3 返回值说明

| 调用形式                | 返回值个数 | 返回类型 | 中文说明             |
|-------------------------|------------|----------|----------------------|
| `Device.wifiAddress()` | 1          | string   | Wi‑Fi MAC 物理地址   |

失败时返回 `nil`。

---

### 14.4 Lua 调用示例

```lua
local wifi = Device.wifiAddress()

if wifi then
    Log.print('------------------------------------')
    Log.print('Wi-Fi MAC 地址: ' .. wifi)

    -- 小技巧：通过 MAC 地址判断厂商信息（前三组）
    local prefix = wifi:sub(1, 8)
    Log.print('网卡前缀标识: ' .. prefix)
    Log.print('------------------------------------')
else
    Log.print('错误: 无法获取 Wi-Fi 地址')
end
```

## 16. `Device.cpuArchitecture()`

### 16.1 接口说明

`Device.cpuArchitecture()` 用于获取设备 **CPU 的指令集架构信息**，  
返回格式示例：`arm64` 或 `arm64e`。

该接口返回中央处理器（CPU）的指令集版本，具体说明如下：  
- `arm64`: 适用于 iPhone 5s 到 iPhone X 的设备。  
- `arm64e`: 适用于 iPhone XS 及以后（A12 芯片及更新）的设备，支持指针自签名加密（Pointer Authentication）。

常用于：  
- 判断设备的硬件特性  
- 针对不同架构的优化  

---

### 16.2 参数说明
